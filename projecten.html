VVD

Vorig jaar heeft Us Media de vernieuwde website van de VVD gebouwd, door de druk van de verkiezingen zijn niet alle wensen vervult voor de eindoplevering. Deze wensen zijn opgenomen in een nieuw project.

Ik heb in dit project kleine frontend aanpassingen doorgevoerd, zoals het toevoegen van extra pagina's en filter functionaliteiten. Mijn belangrijkste werkzaamheden waren echter het toevoegen van een betalingsmogelijkheid via iDEAL. Hiervoor heb ik overleg moeten plegen met de payment provider (ABN Amro) om te bepalen hoe de betalingsberichten eruit moesten zien, in de nieuwe iDEAL versie 3. Nadat ik genoeg informatie heb ingewonnen heb ik een library geschreven in PHP die betalingen mogelijk maakt. Deze library is vervolgens geïntegreerd in de nieuwe website van de VVD.

Desigual image viewer

Desigual had de behoefte om professionele afbeelding en video materiaal in de webshop te zetten. Ze wilden gebruik maken van Adobe Scene7, dat is om hoge resolutie assets te kunnen gebruiken op hun website.

Ik heb de image viewer gemaakt met het GWT framework in Java. Om dit te bereiken heb ik overleg gehad hoe de applicatie geïmplementeerd zou gaan worden in de website.

Toyota & Veilig Verkeer Nederland

Bij de start van het schooljaar gebeuren er meer ongelukken dan later in het jaar. Veilig Verkeer Nederland wilde kinderen en ouders daar extra op attenderen. Samen met Toyota wilde Veilig Verkeer Nederland een actie opzetten, waarbij een sportteam een eigen ontworpen auto kon winnen die de kinderen naar hun sportclub brengt en haalt.

Ik heb de backend applicatie gemaakt met het AmfPHP en Symfony 1 framework in PHP. In de flash frontend applicatie konden kinderen hun auto ontwerpen, door een afbeelding te uploaden, teksten toe te voegen en kleuren van de auto te wijzigen. Deze data werd vervolgens verstuurd naar de backend en werd opgeslagen. Ik heb een CMS gemaakt waarin een selectie van auto's gemaakt kon worden, die meededen aan de nominatie ronden. Via de website konden kinderen stemmen, ook dit werd via de backend opgeslagen.

Tommy Hilfiger Fit guide

Tommy Hilfiger merkte dat klanten vaak broeken retourneerde. Om klanten beter te helpen bij het selecteren van hun broeken, wilden ze klanten een bepaalde broek stijl aanmeten (Fit guide). In overleg met Tommy Hilfiger heb ik de data uitwisseling besproken. Deze data werd gebruikt om de broeken bij de juiste stijl te selecteren. Tevens stuurden wij geselecteerde filters terug naar de Tommy Hilfiger website, zodat bijbehorende broeken direct getoond konden worden. De applicatie moest een single page interface hebben, die boven de huidige webshop zou staan.

Ik heb de frontend applicatie gemaakt met JavaScript, CSS en HTML. Transities van schermen en afbeeldingen heb ik geanimeerd met een JavaScript, om de applicatie visueel aantrekkelijk te maken.

Zylom Challange

Zylom zag het mediaconsumptiegedrag verschuiven naar gaming. Ze wilden daarom een competitie starten namelijk the Zylom Challange. Met the Zylom Challenge zouden mediabureaus dezelfde ervaring moeten krijgen, als consumenten dat al hadden op het Zylom platform.

De backend applicatie van de Zylom Challange heb ik in PHP gebouwd. In afstemming met de frontend, is AMF gekozen als data uitwisselingsmethode. In de backend werden gamesessies en gamescores opgeslagen. Ook konden hier bedrijf- en gebruikeraccounts aangemaakt worden. De game is door 74 bureaus meer dan 5500 keer gespeeld en is daarmee een succes voor Zylom.
